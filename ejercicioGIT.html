Examen Git y Github

Alumn@: JJFS

  1. ¿Que es un sistema de control de versiones?

    Los sistemas de control de versiones son herramientas de software que ayudan a los equipos de software a gestionar los cambios en el código fuente a lo largo del tiempo.

  2. ¿Cual es la diferencia entre git y github?

    Git se utiliza para rastrear y gestionar cambios en el código, mientras que GitHub proporciona una interfaz para compartir ese código y colaborar con otros.

  3. ¿Cómo creo una rama en git?

    El comando git branch permite crear una rama nueva.

  4. ¿Cómo clono un repositorio?

    git clone "url de proyecto que quieras clonar"

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Con el comand git pull te traes los cambios echos en el repositorio(main)

  6. ¿Cómo puedo gener un tag?

    Con git tag -a v1.4 -m 'my version 1.4'

  7. Cómo hago una mezcla de ramas?

    Emplea git merge y especifica el nombre de la otra rama que deseas traer a esta rama.

  8. ¿Cómo puedo moverme a un commit especifico?

    Con git checkout c3

  9. ¿Cómo puedo borrar una rama?

    git branch -d nombre-rama con el parametro -d borariamos la rama

  10. ¿Cómo puedo borrar un tag?

    git tag -d nombre-rama con el parametro -d borariamos el tag

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Con el comadndo git status verias todos los cambiaos realizados

  12. ¿Para que sirve el comando git remote -v?

    Puedes verificar que todo esté funcionando correctamente

  13. ¿Cómo le digo a git cual es mi email y correo?

    git config --global user.name "Tu Nombre" git config --global user.email "tuemail@example.com"

  14. ¿Cuales son los pasos para conectar git con github?

    1. Instalar Git
    2. Configurar Git
    3. Conectar tu repositorio local con GitHub
    4. Subir tu proyecto al repositorio en GitHub
    5. Verificar conexion

  15. ¿Cómo puedo crear un alias en git?

    git config --global alias.nombre-del-alias 'comando'
